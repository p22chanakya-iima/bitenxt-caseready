# BiteNxt CaseReady™

**Scan Verification Engine for Indian Dental Labs**

Context-aware dental preparation STL analysis — marginal gap, occlusal clearance, undercut detection, taper angle. GREEN means mill. RED means rescan.

---

## Local Development

### Prerequisites
- Node.js 20+
- Python 3.9+
- pip

### 1. Install dependencies

```bash
npm install
pip install -r requirements.txt
```

### 2. Generate demo STL files

```bash
python generate_stl_demos.py
```

This creates three synthetic tooth preparation meshes in `public/stl/`:
- `case_a_tooth36_implant.stl` — RED case (undercut + low taper)
- `case_b_tooth14_natural.stl` — GREEN case (clean preparation)
- `case_c_tooth21_anterior.stl` — YELLOW case (borderline)

### 3. Run the frontend

```bash
npm run dev
```

Frontend available at http://localhost:3000

### 4. Test the Python API locally

The API (`api/analyse.py`) is a Vercel serverless function. For local testing:

```bash
# Install vercel CLI
npm i -g vercel

# Run with Vercel dev (serves both Next.js and Python functions)
vercel dev
```

Or test the Python algorithms directly:

```python
import trimesh
from api.analyse import validate_scan_quality, detect_undercuts

mesh = trimesh.load('public/stl/case_a_tooth36_implant.stl', force='mesh')
print(validate_scan_quality(mesh))
print(detect_undercuts(mesh))
```

## Algorithms

### 1. Scan Quality Validation
Checks mesh integrity: watertight status, winding consistency, vertex count, degenerate face ratio, physical dimensions. Outputs 0–100 quality score.

### 2. Undercut Detection
Two methods combined: (a) face normal analysis — lower-half faces pointing upward indicate undercut geometry; (b) ray-casting grid from above — multiple intersections per ray confirm undercuts. Multi-hit ratio > 8% = MODERATE, > 20% = SEVERE.

### 3. Taper Angle Measurement
Isolates axial wall faces (15–85% height zone, mostly-horizontal normals). Computes angle between each face normal and the horizontal plane. Ideal: 4–8° per-half convergence.

### 4. Margin Line Detection
Uses `trimesh.curvature.discrete_mean_curvature_measure` to find high-curvature vertices in the gingival zone (bottom 30% of height). Computes radial regularity score — higher = more circular, cleaner margin.

### 5. Occlusal Clearance Measurement
Requires opposing arch STL. Finds closest points from prep occlusal vertices to opposing mesh. Locates functional cusp (closest opposing cusp to prep center). Minimum clearance requirements: 3Y = 1.5mm, 4Y = 1.2mm, 5Y = 0.7mm. +0.3mm buffer for bruxism.

---

## Limitations

- Marginal gap measurement requires crown STL (not implemented in prototype — approximated from margin regularity)
- Occlusal clearance requires opposing arch STL upload
- Taper angle is computed from face normals, not direct wall measurement
- All scores are geometric proxies; clinical calibration is in progress
- Demo STL files are synthetic — real scanner output will produce different scores
